# Icon：失敗とジェネレータが言語になる

## 真偽値のない言語

Icon [](#cite:griswold1996) は SNOBOL（文字列処理の伝説的言語）の
後継として 1970 年代に生まれました。最大の特徴は、**式の評価が
「値を返す」か「失敗する」かのどちらか**であることです。真偽値は
存在しません。

```icon
if x > 5 then write("big")
# 「x > 5」は true/false を返すのではない。
# 成功すれば「5 より大きいその値 x」を返し、だめなら「失敗」する
```

比較演算子が**値を返す**ので、`3 < x < 10` のような数珠つなぎが
そのまま動きます（`3 < x` が成功すると x を返し、それが `< 10` の
左辺になる）。失敗は偽ではなく「**この道は行き止まり**」という
信号で、評価器はそれを受けて**別の候補を試し**ます。

## すべての式がジェネレータ

「別の候補」はどこから来るのか。Icon では**あらゆる式が潜在的に
ジェネレータ**（generator、複数の結果を順に生み出す装置）だから
です。

```icon
every write(1 to 3)        # 1, 2, 3 を順に書く（ループ構文ではない！）
write((1 to 10) > 7)       # => 8。1..10 を順に試し、最初に 7 を超えた値
```

`1 to 10` は 1 を生み、それが `> 7` で失敗すると、**評価器が
ジェネレータまで巻き戻って次の 2 を再開**します。8 で初めて成功し、
全体の値は 8。これを**目標指向評価**（goal-directed evaluation）と
呼びます。プログラム全体が、正規表現の章で見たバックトラック VM の
ように、また Prolog の章の選択点とトレイルのように動くのです ——
ただし、論理でも正規表現でもなく**ふつうの式の評価**として。

## 実装：中断できる式、再開できるフレーム

データ構造の視点では、Icon の評価器は次を備える必要があります。

- **中断できる式の状態**：`1 to 10` は「いま 7 まで生んだ」という
  状態を保持して眠り、失敗の信号で起こされて 8 を生む。これは
  クロージャの章で見た**ジェネレータのフレーム保存**そのものです。
  Icon ではすべての式がこれを要求するため、式の評価状態（どの
  サブ式まで進んだか）を保持する小さなフレームが評価器の基本単位に
  なります。
- **失敗の伝播経路**：どの式が失敗したら、どのジェネレータを
  再開するか。正規表現エンジンのバックトラックスタック、Prolog の
  選択点スタックと同じ「やり直し台帳」が、言語の評価器の中心に
  座ります。

さらに Icon には **co-expression**（コエクスプレッション）という
機能があり、式を独立した実行文脈（専用スタック付き —— スタックと
フレームの章で見たスタックフル・コルーチン）として切り出し、
`@` 演算子で 1 結果ずつ取り出せます。1970 年代の言語が、現代の
ジェネレータ／コルーチンの全部品を先取りしていたわけです。

## 文字列走査：暗黙のカーソルというデータ

SNOBOL 譲りの**文字列走査**（string scanning）も独特です。

```icon
line ? {                  # line を「被走査文字列」に設定
    tab(upto(":"))        # ":" の手前まで進む
    move(1)               # 1 文字進む
    name := tab(0)        # 残り全部を取る
}
```

`?` の中では、**被走査文字列と現在位置（カーソル）**という暗黙の
状態が維持され、`tab` や `move` がそれを進めたり戻したりします。
そして走査関数も**失敗できる**ので、目標指向評価と組み合わさって、
正規表現に匹敵するパターン照合が**ふつうの式の組み合わせ**で書けます。
正規表現の章で見た「パターンは小さなプログラム」の逆方向 ——
**プログラムをそのままパターンにする**アプローチです。現代の
パーサコンビネータ（Haskell の parsec など、「失敗できる部品を
合成して構文解析器を作る」ライブラリ群）は、この発想の直系と
言えます。

## 系譜と教訓

Icon の影響は、名指しで現代に残っています。Python のジェネレータと
`yield` を導入した PEP 255 は、設計の先行例として Icon を明記して
います [](#cite:pep255)。「成功／失敗」の二値は、現代の `Option`／
`Maybe` 型（Rust の `Option<T>`、値の表現の章のニッチ最適化で
触れた型）や、`?.`（nil 安全呼び出し）演算子の**失敗の伝播**に
形を変えて生きています。

教訓を一つ選ぶなら：**「制御構造」と「データ構造」の境界は思うより
薄い**。Icon は「ループ」「真偽値」「パターン」という制御の語彙を、
「再開できるフレーム」「失敗の台帳」「カーソル」というデータ構造に
置き換えてみせました。本書で見てきた継続・ジェネレータ・バック
トラックスタックは、言語の表面からは見えない位置にいることが多い
けれど、Icon ではそれが言語の顔なのです。

次の章では、統計の世界で独自進化した言語 —— 引数がすべてサンクで、
代入がすべてコピー（のふり）をする R を訪ねます。
